# Lynne 需求文档 v3.0

> 先说清楚输入和输出，再谈怎么实现。

---

## 1. 产品定义

Lynne 是一款**由大模型驱动的个人社交平台信息自动化采集与分析工具**。在你自己电脑上运行，通过 Web 页面操作，让非技术用户也能轻松使用。

**核心理念**：
- **浏览器自动化 = 手脚**：去各平台帮你把原始数据拿回来
- **大模型 = 大脑**：理解内容、判断相关性、提取要点、生成洞察
- **Web 界面 = 面孔**：像网页应用一样使用，不需要敲命令
- **JSON 文件 = 存储**：数据直接存文件，可以直接打开看，不需要装数据库

**一句话**：打开一个网页，告诉它你关心什么 → 它去各平台搜回来 → 用 AI 理解筛选总结 → 在网页上给你看结果。

### 2.2 辅助方式：配置文件

设置项通过 Web 页面操作，底层存为 `config.yaml`，也可以直接编辑：

```yaml
llm:
  provider: "deepseek"
  api_key: "${DEEPSEEK_API_KEY}"

tasks:
  - name: "AI行业动态"
    platforms: [twitter, rednote]
    topic: "关注AI大模型的最新进展"
    schedule: "every 4 hours"
    limit: 20
```

### 2.3 CLI 保留（可选）

技术用户仍然可以通过命令行操作，但不作为主要交互方式：

```bash
lynne serve          # 启动 Web 服务（直接打开浏览器）
lynne run            # 直接运行一次采集
lynne status         # 查看状态
```

## 3. 输入（Input）

### 3.1 Web 页面上用户提供什么

| 输入项 | 在哪里配置 | 示例 |
|--------|-----------|------|
| **平台账号** | "登录" 页面，点击各平台手动登录 | 小红书扫码、Twitter 账号密码 |
| **关注主题** | "配置" 页面，自然语言描述 | "我想了解 AI 大模型的最新进展" |
| **目标账号** | "配置" 页面，输入要追踪的账号 | `@OpenAI`, `@karpathy` |
| **平台选择** | "配置" 页面，勾选 | ☑ Twitter ☑ 小红书 ☐ 抖音 |
| **采集频率** | "配置" 页面，下拉选择 | 每4小时 / 每天8点 / 手动 |
| **数量限制** | "配置" 页面，数字输入 | 每个平台每次 20 条 |
| **LLM 设置** | "设置" 页面 | 选 provider + 填 API Key |

### 3.2 输入 vs 传统关键词搜索

```
传统方式（纯规则）：
  输入: ["AI", "大模型", "AGI"]
  → 只能匹配字符串，大量漏判误判

Lynne 方式（大模型理解）：
  输入: "关注AI大模型领域的技术突破和产品发布"
  → LLM 理解你的真实意图
  → 能发现没用到关键词但实际高度相关的内容（语义匹配）
  → 能区分 "讨论GPT-5论文"（相关）和 "跟AI瞎聊天"（不相关）
```

## 4. 大模型在工具中的角色

```
浏览器采集（原始数据）
       │
       ▼
① LLM 相关性筛选
  判断每条内容与你的关注主题相关吗？打分1-10，低于阈值丢弃
       │ 只保留相关的
       ▼
② LLM 内容理解
  每条内容生成一句话摘要 + 话题标签 + 关键信息点
       │
       ▼
③ 存入文件（JSONL）
  原文 + LLM 富化数据一起保存
       │
       ▼
④ LLM 日报生成
  把当天所有留存数据归纳成结构化日报，按话题聚类
       │
       ▼
  Web 页面上展示
```

| 环节 | 大模型做什么 | 没有大模型会怎样 |
|------|------------|----------------|
| **① 筛选** | 读帖文，判断是否跟你的关注主题真正相关 | 关键词匹配 → 大量噪音 + 漏掉 |
| **② 理解** | 给每条内容一句话摘要，打上话题标签 | 40条原文全部摊给你，自己一条条看 |
| **③ 关联** | 发现不同帖子在讨论同一件事（跨平台） | 碎片化信息，看不出关联 |
| **④ 报告** | 把40条碎片总结成3-5个关键进展 | 没有"所以呢？"这一层 |

### 分层调用策略

```
筛选层：便宜模型（DeepSeek/deepseek-chat / gpt-4o-mini）
  → 每条调1次，量最大但 prompt 短，成本低

摘要层：同上便宜模型
  → 每条调1次，生成摘要和标签

日报层：强模型（deepseek-chat / gpt-4o）
  → 每天调1次，汇总所有留存内容生成日报
```

## 5. 输出（Output）

### 5.1 输出内容

| 输出项 | 内容 | 在哪看 |
|--------|------|--------|
| **智能日报** | AI 生成的今日关键话题汇总 | Web 首页 / 日报页 |
| **帖子详情** | 正文 + LLM摘要 + 标签 + 互动数据 | Web 详情页 |
| **相关度评分** | LLM 判断每条内容的匹配度 1-10 | 每条帖子卡片上 |
| **话题标签** | LLM 自动打的标签 | 详情页 + 可筛选 |
| **历史回顾** | 按日期/话题回看过去的日报和数据 | "历史" 页面 |
| **采集状态** | 采集了多少条、筛后留存多少、耗时 | Web 顶栏 / 日志页 |

### 5.2 数据存储形式

```
data/
├── 2026-04-25/                  # 每天一个目录
│   ├── items.jsonl              # 采集的原始+富化数据，一行一条
│   ├── report.md                # 日报 Markdown
│   └── summary.json             # 任务执行摘要
├── 2026-04-24/
│   └── ...
└── sessions/                    # 浏览器会话（免重复登录）
    ├── twitter_state.json
    ├── rednote_state.json
    └── douyin_state.json
```

**items.jsonl 示例（每行一条）：**
```json
{"platform":"twitter","item_id":"189123456","author_name":"@OpenAI","content_text":"We are excited to announce GPT-5...","url":"https://x.com/OpenAI/...","published_at":"2026-04-25T10:30:00Z","metrics":{"likes":45200,"comments":3100,"shares":8900},"llm_relevance_score":9,"llm_relevance_reason":"直接讨论了GPT-5发布","llm_summary":"OpenAI正式发布了GPT-5，主打推理增强和多模态实时交互","llm_tags":["#模型发布","#多模态","#推理"],"llm_key_points":["推理能力提升2倍","支持实时视觉交互","价格未公布"]}
```

## 6. Web 界面功能

### 6.1 页面结构

| 页面 | 功能 |
|------|------|
| **首页/日报** | 显示最新日报，概述、话题聚类、来源帖子列表 |
| **配置页** | 添加/编辑采集任务（主题、平台、关键词、频率） |
| **登录页** | 触发各平台手动登录流程，查看会话状态 |
| **历史页** | 按日期浏览历史数据和日报 |
| **搜索页** | 搜索本地已采集数据 |
| **设置页** | LLM 配置（provider、API Key、模型选择） |
| **日志页** | 查看采集运行日志 |

### 6.2 典型操作流程

```
1. 第一次使用：
   打开 http://localhost:7890 
   → 进入设置页，配置 LLM（选 provider + 填 key）
   → 进入登录页，逐个平台手动登录
   → 进入配置页，添加第一个任务（选平台、写主题、定频率）
   → 回到首页，点"立即采集"
   → 等待几分钟，看日报生成

2. 日常使用：
   打开页面 → 看日报 → 感兴趣的点详情 → 完事

3. 临时需求：
   Web 页有"手动搜索"输入框，输入想了解的话题，即时采集
```

## 8. 非功能需求

| ID | 需求 | 说明 |
|----|------|------|
| NF1 | **本地运行** | 数据不经过第三方服务器（LLM API 除外） |
| NF2 | **不封号** | 操作频率模拟人类行为 |
| NF3 | **容错性** | 某平台挂了不影响其他；LLM 挂了保留原始数据 |
| NF4 | **隐私安全** | 会话文件不进 git；API Key 走环境变量 |
| NF5 | **简单部署** | 装好依赖后一个命令启动，自动打开浏览器 |
| NF6 | **无需数据库** | 数据存文件，每天一个目录，直接可看 |
| NF7 | **成本可控** | 筛选用便宜模型，日报用强模型 |

## 10. 约束与假设

| 项 | 说明 |
|----|------|
| **运行环境** | 你自己的电脑，Linux/Mac |
| **LLM 依赖** | 核心功能强依赖 LLM；不可用时降级为原始数据输出 |
| **成本** | API 方案每日 100-200 条 ≈ $0.3-$1/天；本地 Ollama 零成本 |
| **不在范围内** | 不搞代理池、多账号、大规模采集、对外服务 |

---

## 11. 成功标准

1. 装好依赖 → 一个命令启动 → 浏览器自动打开 Web 页面
2. 在 Web 上配好任务 → 点采集 → 几分钟后看到 AI 生成的日报
3. 日报只含 3-5 个关键话题，不是 40 条碎片
4. 没有无关帖子混进日报
5. 跑一周，账号没被封
6. 非技术用户也能独立完成配置和使用
