# core/browser 测试方案

## 1. 被测目标

| 源文件 | 类型 | 测试文件 | 说明 |
|--------|------|---------|------|
| `browser_models.py` — `BrowserConfig` | UT | `test_browser_models.py` | dataclass 默认值、自定义、相等性 |
| `browser_factory.py` — `BrowserManagerFactory` | UT | `test_browser_factory.py` | create 行为、类型返回 |
| `browser_manager.py` — ABC 接口 | — | 不单独测试 | ABC 无实例可测 |
| `imp/playwright_context.py` — `PlaywrightContext/Page` | — | 不单独测试 | 通过 BrowserManager TA 间接覆盖 |
| `imp/playwright_browser_manager.py` | **TA** | `test_browser_impl.py` | 真实 Playwright headless 全链路 |

> TA 测试启动真实 Chromium（headless），使用 `BrowserManagerFactory` 创建实例，`tmp_path` 隔离会话文件。验证完整的 start → context → page → session → stop 生命周期。

---

## 2. 测试环境

| 依赖 | 说明 |
|------|------|
| Chromium | Playwright 安装的 `/root/.cache/ms-playwright/chromium-*/chrome-linux64/chrome` |
| headless | TA 中 `BrowserConfig(headless=True)`，无需显示器 |
| `tmp_path` | pytest 内置 fixture，隔离会话文件，测试结束自动清理 |
| `pytest-asyncio` | 所有 TA 用例标记 `@pytest.mark.asyncio` |

运行命令：
```bash
# 仅 TA（需要 Chromium）
pytest tests/core/browser/test_browser_impl.py -v -m asyncio

# 全部 browser 测试
pytest tests/core/browser/ -v
```

## 4. TA 用例详细设计

### 4.1 完整生命周期

```
ta_lifecycle fixture:
  BrowserManagerFactory().create(
    BrowserConfig(headless=True, sessions_dir=tmp_path / "sessions")
  )

test_full_lifecycle:
  await manager.start()
  assert manager.health_check() is True         ← Chromium 真实在运行
  ctx = await manager.get_context("twitter")
  page = await ctx.new_page()
  await page.goto("data:text/html,<h1>Lynne</h1>")
  await manager.save_session("twitter")
  await manager.stop()
  assert manager.health_check() is False        ← 进程已退出
  assert (tmp_path / "sessions" / "twitter_state.json").exists()
```

### 4.2 Session 往返

```
ta_session_roundtrip:
  第一轮：manager_A.start() → get_context → save_session → stop
  第二轮：manager_B.start() → get_context("twitter")
    → 验证 context 创建成功（session 文件被加载）
    → 验证 context.raw().pages[0] 或类似属性可用
    → stop
```

### 4.3 Context 隔离

```
ta_context_isolation:
  start
  ctx1 = get_context("twitter")
  ctx2 = get_context("rednote")
  assert ctx1 is not ctx2
  assert ctx1.raw() is not ctx2.raw()
```

### 4.4 Login Flow

```
ta_login_flow_e2e:
  start
  await login_flow("test_platform", "data:text/html,<title>Login</title>")
  assert manager._login_pending["test_platform"] is True
  await set_login_complete("test_platform")
  assert "test_platform" not in manager._login_pending
  assert session file exists
```

## 5. 不覆盖的边界

| 场景 | 原因 |
|------|------|
| 真实平台登录（twitter.com / xiaohongshu.com） | 需要真实账号，属于集成测试范畴 |
| 多并发 context 操作 | 单模块 TA 不测并发，属于更高层 IT |
| Playwright 进程崩溃恢复 | Playwright 自身保证，不属于本模块职责 |
