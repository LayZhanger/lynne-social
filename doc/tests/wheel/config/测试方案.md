# wheel/config 测试方案

## 1. 被测目标

| 源文件 | 类型 | 测试文件 |
|--------|------|---------|
| `config_models.h` — `Config` / `LLMConfig` / `ServerConfig` 等 struct | UT | `tests/wheel/config/test_config_models_ut.cpp` |
| `config_factory.h` — `ConfigLoaderFactory` | UT | 同上 |
| `imp/json_config_loader.h/.cpp` — `JsonConfigLoader` | TA | `tests/wheel/config/test_config_loader_ta.cpp` |

## 2. 测试点清单

### UT: 数据模型
- [x] struct 默认值 — port=7890, provider=deepseek, model=deepseek-chat
- [x] from_json 解析 — 部分字段、完整字段
- [x] 枚举类型 — LogLevel 值互异

### UT: 工厂
- [x] `create()` — 返回 JsonConfigLoader
- [x] 返回类型是 `ConfigLoader` 子类

### TA: JsonConfigLoader 全链路
- [x] 文件不存在 → `Config()` 默认值
- [x] 完整 JSON → 各字段正确解析
- [x] `reload()` — 修改文件后重新加载
- [x] env 变量回退 — `DEEPSEEK_API_KEY`

## 3. 运行命令

```bash
dist/bin/test_config_models_ut
dist/bin/test_config_loader_ta
```
