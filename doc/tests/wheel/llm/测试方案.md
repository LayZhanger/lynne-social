# wheel/llm 测试方案

## 1. 被测目标

| 源文件 | 类型 | 测试文件 |
|--------|------|---------|
| `llm_models.py` — `LLMConfig` | UT | `tests/wheel/llm/test_llm_models.py` |
| `llm_factory.py` — `LLMEngineFactory` | UT | `tests/wheel/llm/test_llm_factory.py` |
| `imp/deepseek_engine.py` — `DeepSeekEngine` | UT | `tests/wheel/llm/test_llm_impl.py` |

> 说明：impl 测试使用 `unittest.mock.patch` 替换 `urllib.request.urlopen`，全程无真实网络调用。真实 API 调用由 E2E 覆盖（将来）。

## 2. 测试点清单

### UT: LLMConfig (4 例)

- [x] 默认值 — provider="deepseek", model="deepseek-chat", temperature=0.7, max_tokens=4096, timeout=60
- [x] 自定义值 — 各字段均可覆盖
- [x] 相等 — 同值 dataclass 相等
- [x] 不等 — 不同 api_key 不相等

### UT: 工厂 (4 例)

- [x] `create(LLMConfig(api_key="sk-test"))` — 返回 DeepSeekEngine，name 包含提供者和模型
- [x] `create(LLMConfig(...))` — 返回 LLMEngine 接口子类
- [x] 缺 api_key — `create(LLMConfig(api_key=""))` 抛出 ValueError
- [x] 自定义 model — name 中显示 "deepseek-reasoner"

### UT: DeepSeekEngine (15 例)

#### 生命周期

- [x] `start` + `stop` — name 正确，启停不抛异常
- [x] `health_check` 无网络 — 返回 False

#### chat 正常流程

- [x] `chat` 纯文本 — 返回 `{"role":"assistant","content":"...","tool_calls":null}`
- [x] `chat` 含 tool_calls — 返回 tool_calls 列表（id, type, function.name, function.arguments）
- [x] `chat` 工具参数透传 — 请求体中 tools 字段正确传入
- [x] 自定义 base_url — 请求 URL 使用自定义地址
- [x] 自定义 temperature/max_tokens — 请求体中参数正确

#### 错误处理

- [x] HTTP 401 — 抛出 RuntimeError("LLM API returned 401: ...")
- [x] HTTP 500 且 body 不可读 — 只报 status code，不因 body 读取失败而崩溃
- [x] URLError（连接拒绝/DNS） — 抛出 RuntimeError("LLM API unreachable: ...")

#### 名称

- [x] `name` 反映 provider 和 model

## 3. 公共依赖

- `unittest.mock.patch` — 替换 `urllib.request.urlopen`，模拟 API 响应
- `LLMConfig` — 构造测试配置
- `LLMEngineFactory` — 工厂创建
- `DeepSeekEngine` — 被测实例

## 4. 运行命令

```bash
pytest tests/wheel/llm/ -v
```
